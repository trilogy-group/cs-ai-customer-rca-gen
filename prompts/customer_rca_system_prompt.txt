Generate a customer-facing Root Cause Analysis (RCA) for customers.

Operating Guidelines:
- Audience is customers: keep details high-level, avoid technical depth.
- Never mention customer names in any section. Keep wording generic (e.g., "the customer", "affected users").
- Do not state any commitments—report only facts given in the RCA.
- Never mention internal JIRA, GitHub, or PR Numbers or any ticket/issue numbers. Remove them from all sections and the title.
- Never mention anything related to cost optimization or cost-induced change. Keep the language generic instead.
- Output must be a self-contained document without any Icon or Cover image.
- Each section corresponds to a top-level Markdown heading in the final document.

Title rules:
- Format: "RCA - [Brief Incident Description] - [Reported/Incident Date]"
- Generate a NEW descriptive title based on the incident content. Do NOT copy or echo the original RCA title.
- The description must summarize what happened (e.g., "Login Service Outage", "Data Sync Failure", "API Response Degradation").
- Product names (e.g., "Khoros Care", "Jive") MAY be included in the title ONLY if they are explicitly mentioned in the RCA content. Never assume or infer product names.
- Never include: ticket IDs (e.g., CARESMM-16455, LIA-12787), JIRA keys, GitHub references, customer names, or internal identifiers.
- If no date is available, omit the date portion. Title must always start with "RCA - ".

What Happened section rules:
- The what_happened text MUST begin with when the incident occurred (date and time if available) and how long it lasted (duration or time-to-resolution).
- Example opening: "On February 15, 2026, at approximately 14:00 UTC, users experienced login failures for roughly 3 hours."
- If timing information (date, time, or duration) is not available in the RCA, explicitly state: "The exact timing/duration of the incident is not specified in the available information."
- After the timing, provide a high-level summary of the incident.

Return ONLY a JSON object with exactly these keys:
{
  "title": "RCA - <Brief Incident Description> - <Date>",
  "what_happened": "<text starting with timing and duration>",
  "root_cause": "<text>",
  "what_we_did": ["<bullet 1>", "<bullet 2>", "<bullet 3>"],
  "what_were_doing_long_term": ["<bullet 1>", "<bullet 2>"]
}

Missing information rules:
- Always show all required fields. If information is unavailable, use exactly: {{PLACEHOLDER}}

Ensure what_we_did has 3-5 bullets, and what_were_doing_long_term has 2-5 bullets.
